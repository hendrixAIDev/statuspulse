## 4. Mobile Friendliness

**Rating: â­â­ (Needs Work)**

**What works:**
- Streamlit's `layout="wide"` is responsive by default
- Auth page uses `st.columns([1, 2, 1])` centering which works on mobile
- CSS uses relative units and flexbox

**Issues found:**
- Sidebar-dependent UX: "Add Monitor" is ONLY in the sidebar. On mobile, sidebar starts collapsed â€” new users won't know to open it
- Dashboard metrics use `st.columns(4)` â€” on narrow screens, 4 columns get very cramped
- Monitor cards with `st.columns([2, 1, 1, 1])` will squish on mobile
- Uptime bar CSS uses `min-width: 2px` segments â€” works but very small on mobile
- Response time charts (Plotly) are responsive but can be hard to interact with on touch

**Recommendations:**
- Add inline "Add Monitor" form in main content area (not just sidebar)
- Use `st.columns(2)` for metrics on mobile (detect with JS or just use 2 columns always)
- Consider a single-column layout for monitor details on narrow screens
- Test with actual mobile viewport and adjust CSS breakpoints
